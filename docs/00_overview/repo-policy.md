# リポジトリ運用ポリシー

## このファイルの役割

このリポジトリを長く使うための最小ルールをまとめたメモです。新しい Markdown や資料を追加するときは、まずこの方針に沿っているかを確認します。

## 基本方針

- このリポジトリは資料置き場ではなく、再利用できる知識ベースとして育てる
- 新しい情報はまず Markdown で整理し、PDF や画像は補助資料として扱う
- 1トピック1ファイルを基本にして、後から検索しやすい形を優先する
- 迷いやすい論点は、要約ではなくチェックリストや判断ポイントとして残す

## 追加時のルール

- docs 配下の適切なカテゴリに置く
- ファイル名は ASCII の kebab-case にする
- 税務や制度の話は、対象年度、最終確認日、参照元をできるだけ書く
- PDF や画像を追加する場合は、それを説明する Markdown も用意する

## 保存しない情報

- マイナンバー
- 実在する契約書、請求書、申告書の原本
- 取引先名や金額が分かるスクリーンショット
- アカウント、パスワード、API キーなどの認証情報

## 見直しの目安

- 税務情報は申告時期の前に再確認する
- 制度改定があったら旧情報を archive 候補として見直す
- README と docs/repository-structure.md の導線が崩れていないか定期的に確認する
