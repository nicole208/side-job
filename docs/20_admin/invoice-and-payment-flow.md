# 請求から入金確認までの流れ

## このメモの役割

副業でありがちな「納品したのに請求を忘れる」「請求したのに入金確認を後回しにする」を防ぐための運用フローです。案件単位の管理ではなく、毎月同じ型で回すことを目的にしています。

## 基本フロー

1. 作業完了条件を確認する
2. 請求に必要な情報をそろえる
3. 請求書を作成して送付する
4. 送付日と支払期日を記録する
5. 入金予定日以降に着金確認をする
6. 未入金なら確認連絡を入れる
7. 入金済みに更新して証憑を保存する

## 請求前チェックリスト

- [ ] 契約や発注内容に基づく請求条件を確認した
- [ ] 金額、数量、期間、源泉徴収の有無を確認した
- [ ] 請求先名義、宛名、送付方法を確認した
- [ ] 支払期日を確認した
- [ ] 請求書番号を付番した

## 管理表に残す最低項目

- 請求書番号
- 取引先名
- 対象期間
- 請求金額
- 請求日
- 支払期日
- 入金予定日
- 実際の入金日
- ステータス

## ステータス例

- draft
- sent
- waiting-payment
- paid
- follow-up-needed

## 請求書送付時のメモ

- メール送付なら件名を毎回似た形にそろえる
- 送付本文には対象期間、金額、支払期日を明記する
- 添付ファイル名は後から検索しやすい形にする

ファイル名例:

- 2026-001-invoice-client-a.pdf
- 2026-002-invoice-client-b.pdf

## 未入金時の対応

- まず管理表の支払期日と口座入金を再確認する
- 期日経過後の何営業日で連絡するか、自分の基準を決めておく
- 連絡履歴は管理表か案件メモに残す
- 感情的な文面にせず、事実確認として送る

## 判断ポイント

- 取引先側で請求書フォーマット指定があるか
- 源泉徴収や手数料控除が発生するか
- 適格請求書発行事業者番号の記載が必要か
- 請求書ではなく支払明細やプラットフォーム決済で代替されるか

## 月末確認チェックリスト

- [ ] sent のまま長期間止まっている請求がない
- [ ] waiting-payment の案件で入金確認漏れがない
- [ ] paid の案件に入金日が入っている
- [ ] 請求書データと送付履歴が残っている
