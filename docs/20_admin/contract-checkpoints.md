# 契約確認のチェックポイント

## このメモの役割

副業案件を受ける前に、最低限どこを確認しておくべきかを整理するためのメモです。契約内容は案件ごとに異なるため、このファイルは法的結論を出すためではなく、見落としを減らすための確認用として使います。

## 先に見るポイント

- 誰と契約するのか
- 何をどこまで提供するのか
- いくらで、いつ支払われるのか
- 修正対応や追加対応の範囲はどこまでか
- 成果物や知的財産権の扱いはどうなるか
- 秘密保持義務や競業制限があるか

## 契約前チェックリスト

- [ ] 契約当事者の名称が正しい
- [ ] 業務内容と成果物の範囲が具体的に書かれている
- [ ] 納期や作業期間が明記されている
- [ ] 報酬額、支払期日、支払方法が書かれている
- [ ] 源泉徴収や手数料控除の扱いが分かる
- [ ] 修正回数や追加作業の扱いが分かる
- [ ] 中途解約時の扱いが確認できる
- [ ] 知的財産権や利用許諾の扱いが分かる
- [ ] 秘密保持条項に無理がない
- [ ] 本業との両立上、問題になる条項がない

## 特に詰まりやすい論点

### 業務範囲

「デザイン一式」「開発対応一式」のような広い表現だけだと、後で認識違いが起きやすくなります。納品物、対応範囲、除外範囲を言葉にしておく方が安全です。

### 修正対応

修正回数の上限、軽微修正の定義、追加料金の条件が曖昧なままだと、想定外の工数が発生しやすくなります。

### 報酬支払

月末締め翌月末払いなのか、検収後支払いなのか、請求書必須なのかを最初に確認します。

### 権利関係

成果物を納品した時点で権利が全て移転するのか、一定の利用許諾なのかで前提が変わります。再利用可否にも関わるため、曖昧なまま進めない方が安全です。

## 自分用メモに残すとよい項目

- 契約相手
- 契約開始日
- 業務内容
- 納品物
- 金額
- 請求条件
- 修正条件
- 関連ファイル保存先

## 判断ポイント

- 契約書がなく発注メッセージだけで進めていないか
- 口頭認識だけで決めている点が残っていないか
- 本業の就業規則や秘密保持と衝突しないか
- 将来トラブルになりやすい条項が曖昧なままではないか

## 注意

契約や法務の扱いは個別事情で大きく変わります。このメモは一般的な確認用であり、具体的な契約判断が必要な場合は契約書本文を優先し、必要に応じて専門家確認を検討します。
